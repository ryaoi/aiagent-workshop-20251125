# AIとAIエージェント入門

## AIとは？

AI（人工知能）は、人間のように学習し、推論し、問題を解決できるコンピュータシステムです。

現代のAIの多くは**大規模言語モデル（Large Language Model, LLM）**を使用しています。GPT-4、Claude、Geminiなどが代表的なモデルで、大量のテキストデータから学習することで、私たちと自然な言葉で会話できるようになりました。質問に答えたり、文章を生成したり、プログラムのコードを書いたりすることもできます。

例えば、こんな会話ができます：

```
ユーザー：「Pythonで文字列を反転する方法は？」
AI：「text[::-1]を使うと簡単に反転できます。」
```

## AIエージェントとは？

では、**AIエージェント**とは何でしょうか？通常のチャットボットとは何が違うのでしょう。

通常のチャットボットは、質問に答えることはできますが、外部の情報にアクセスしたり、自分で行動を起こしたりすることはできません。一方、AIエージェントは質問を理解するだけでなく、必要なツールを自分で選択し、それを使って情報を取得したり計算したりできます。そして、得られた結果をまとめて答えてくれるのです。

具体的な例を見てみましょう：

```
ユーザー：「東京の天気を教えて。気温が20度以上なら散歩に行く確率を計算して」

AIエージェント：
1. 【思考】天気情報が必要だ → 天気ツールを使おう
2. 【行動】weather: Tokyo
3. 【観察】東京は晴れ、気温22度
4. 【思考】22度は20度以上だ → 確率を計算しよう
5. 【行動】calculate: 0.8 * 100
6. 【観察】80
7. 【回答】東京は晴れで22度です。散歩に行く確率は80%です。
```

このように、AIエージェントは自分で考えながら、複数のツールを組み合わせて問題を解決していきます。

## ReActパターンとは？

このようなAIエージェントを実現する方法の一つが**ReActパターン**です。ReActは「**Reasoning（推論）+ Acting（行動）**」を組み合わせた言葉で、AIが思考と行動を繰り返しながら問題を解決していくアプローチです。

ReActパターンでは、AIは次のようなサイクルで動作します：

```
┌─────────────┐
│   質問      │
└──────┬──────┘
       ↓
┌─────────────────────────────┐
│  Thought（思考）            │
│  「何をすべきか考える」      │
└──────┬──────────────────────┘
       ↓
┌─────────────────────────────┐
│  Action（行動）              │
│  「ツールを使う」            │
└──────┬──────────────────────┘
       ↓
┌─────────────────────────────┐
│  PAUSE（一時停止）          │
│  「ツールの実行を待つ」      │
└──────┬──────────────────────┘
       ↓
┌─────────────────────────────┐
│  Observation（観察）        │
│  「ツールの結果を見る」      │
└──────┬──────────────────────┘
       ↓
    必要なら繰り返す
       ↓
┌─────────────────────────────┐
│  Answer（回答）              │
│  「最終的な答えを出す」      │
└─────────────────────────────┘
```

実際の動きを見てみましょう。簡単な計算の例です：

**質問：** 「15 × 23 は？」

```
Thought: 計算が必要だ
Action: calculate: 15 * 23
PAUSE

Observation: 345

Answer: 15 × 23 = 345 です。
```

もう少し複雑な例も見てみましょう：

**質問：** 「東京の天気は？雨なら傘が必要か教えて」

```
Thought: まず天気を調べる必要がある
Action: weather: Tokyo
PAUSE

Observation: 東京は雨、気温18度

Thought: 雨が降っているので傘が必要だと判断できる
Answer: 東京は雨です。傘を持って出かけることをお勧めします。
```

このように、AIは自分で必要な情報を集めて、それをもとに判断を下すことができるのです。

## 今日学ぶこと

このワークショップでは、段階的にAIエージェントの構築方法を学んでいきます。

まず、OpenRouter（Claude Sonnet 4.5）を使ったAIとの会話プログラムから始めます。次に、システムプロンプトを使ってAIの振る舞いを制御する方法を学びます。そして最後に、ReActパターンを実装して、実際にツールを使えるAIエージェントを作成します。

ワークショップは2段階構成になっています：

1. **シンプルなエージェント（05）**: まず1つのツール（シェルコマンド実行）だけを持つエージェントで、ReActパターンの基本を理解します
2. **高度なエージェント（06）**: 次に複数のツールを持つエージェントで、AIが状況に応じてツールを選ぶ様子を観察します

ワークショップの終わりには、計算、天気情報、メモの保存・読み込み、さらにはシェルコマンド実行まで行える高度なAIエージェントが完成します。AIが質問の内容を理解して、適切なツールを自分で選んで使う様子を見ることができます。そして、そこから自分だけのツールを追加していくこともできるようになるでしょう。
